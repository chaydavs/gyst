# Agent Adapters: Multi-Agent Context File Export

## Overview

`gyst export-context` generates static context files for every AI coding agent detected in the project environment. This allows agents that do not support the Model Context Protocol to still receive team knowledge — rules, conventions, decisions, and known error patterns — without any manual copy-paste.

The command is idempotent: re-running it updates only the Gyst-managed section of each file.

---

## Per-Agent File Specifications

| Agent | File | Format | Idempotent? |
|-------|------|--------|-------------|
| Claude Code | `CLAUDE.md` (project root) | Managed section wrapped in HTML comment markers | Yes — replaces only the marked section |
| Cursor | `.cursorrules` (project root) | Instructional plain-text header + sections | Full overwrite |
| Codex CLI (OpenAI) | `AGENTS.md` (project root) | ## Rules / ## Conventions / ## Architecture | Full overwrite |
| Windsurf | `.windsurfrules` (project root) | Same format as `.cursorrules` | Full overwrite |
| Gemini CLI / fallback | `CONTEXT.md` (project root) | Generic markdown with all sections | Full overwrite |

---

## Content Sections

Every generated file contains up to four sections, drawn from the knowledge base:

| Section | Source entries | Limit |
|---------|---------------|-------|
| Critical Rules | `ghost_knowledge` (all active) | No limit — always included in full |
| Known Error Patterns | `error_pattern` (by source count) | Top 5 |
| Conventions | `convention` (by confidence, threshold 0.4) | Top 15 |
| Architecture / Decisions | `decision` (by recency) | Last 10 |

### Token Budget

A 16,000-character cap is enforced with progressive truncation. If the total content exceeds the cap, each stage below is applied in order until the budget is met:

1. Truncate `error_pattern` content to 150 chars each
2. Reduce error patterns to top 3
3. Truncate `convention` content to 150 chars each
4. Reduce conventions to top 10
5. Reduce decisions to top 5

Ghost knowledge is never truncated.

---

## CLAUDE.md: Marker-Based Idempotency

Claude Code's `CLAUDE.md` receives special treatment because it typically contains human-authored project instructions that must not be overwritten.

The Gyst section is wrapped in HTML comment markers:

```
<!-- BEGIN GYST CONTEXT -->
## Gyst Team Knowledge

_Auto-generated by Gyst. Do not edit this section manually._

### Critical Rules
...

### Conventions
...
<!-- END GYST CONTEXT -->
```

**Update behavior:**
- If both markers are present: the content between them (inclusive) is replaced. Everything outside the markers is unchanged.
- If markers are absent but the file exists: the marked block is appended to the end.
- If the file does not exist: it is created with just the marked block.

This means human-written content in `CLAUDE.md` is always preserved regardless of how many times `export-context` is run.

## Format Options

```bash
# Auto-detect all agents on this machine and write appropriate files
gyst export-context

# Write a specific format
gyst export-context --format claude
gyst export-context --format codex
gyst export-context --format cursor
gyst export-context --format windsurf
gyst export-context --format gemini

# Write all five formats at once
gyst export-context --format all

# Preview output without writing files
gyst export-context --dry-run
gyst export-context --format cursor --dry-run
```

---

## Agent Detection

When `--format` is omitted or `auto`, `export-context` calls `detectEnvironment()` (the same function used by `gyst install`) to identify which agents are present on the machine. Detection checks for tool-specific directories:

| Agent | Detection signal |
|-------|-----------------|
| Claude Code | `~/.claude/` exists |
| Cursor | `~/.cursor/` exists |
| Codex CLI | `~/.codex/` exists |
| Windsurf | `~/.codeium/windsurf/` exists |
| Gemini CLI | `~/.gemini/` exists |

If no agent is detected, `CONTEXT.md` is written as a fallback.

---

## Auto-Regeneration

`gyst export-context` runs automatically at two points:

1. **After `gyst install`** — immediately after hooks are installed, so every detected agent already has a context file before the first session.
2. **After `gyst init`** — at the end of the bootstrap flow, after the KB has been seeded with initial knowledge.

After either command, you can re-run `gyst export-context` manually at any time to refresh the files as your knowledge base grows.

## Implementation

Source: `src/cli/commands/export-context.ts`

Key exported functions:

| Function | Purpose |
|----------|---------|
| `buildContextPayload(db)` | Queries KB, applies token-budget heuristic, returns typed payload |
| `formatClaudeMd(payload)` | Generates the CLAUDE.md Gyst section |
| `formatAgentsMd(payload)` | Generates AGENTS.md (Codex CLI format) |
| `formatCursorRules(payload)` | Generates .cursorrules / .windsurfrules |
| `formatGeneric(payload)` | Generates CONTEXT.md fallback |
| `writeContextFile(path, content, markers?)` | File writer with optional marker-based replace |
| `generateContextFiles(db, projectDir, agents)` | Orchestrates payload + specs + writes |
| `runExportContext(opts)` | CLI action: auto-detect or explicit format |
