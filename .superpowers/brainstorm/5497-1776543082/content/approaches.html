3 Approaches to a Self-Documenting KB

Pick the foundation — we'll layer the automation on top

A

Ghost Knowledge Bootstrap

One-time gyst self-document writes ghost knowledge entries for every major module. Ghost entries = infinite confidence, always surface in tier 0 regardless of BM25 score. Mid-session and git hooks do lightweight updates.

Pros

  • Guaranteed retrieval — ghosts always surface
  • Zero false negatives on critical architecture
  • Minimal token overhead per recall

Cons

  • Ghost entries never decay — stale info persists
  • Quality depends on static analysis accuracy
  • Manual cleanup needed when architecture changes
B

Q&A Corpus + Semantic Anchoring ★ Recommended

Auto-generate a canonical question corpus ("How does the search pipeline work?", "Where is authentication handled?"). Entries stored as learning type, confidence 0.95, graph-linked to module ghosts. Titles match exactly how agents phrase recall() queries.

Pros

  • Natural language overlap with agent query patterns
  • Confidence decay handles staleness automatically
  • Graph links guarantee traversal to related context
  • Anchor queries double as KB health probes

Cons

  • Question set needs maintenance as codebase evolves
  • LLM call required for initial generation
  • Hit rate depends on question phrasing quality
C

Living Module Registry (Graphify-first)

Promote gyst's existing Graphify AST structural nodes directly into KB entries. Each module gets a structural summary auto-derived from its exports, imports, and git blame. No LLM needed for generation.

Pros

  • Builds on existing Graphify infrastructure
  • No LLM tokens for generation — pure AST analysis
  • Always structurally accurate (reflects actual code)

Cons

  • Captures WHAT, not WHY — agents need both
  • Structural language ≠ natural query language
  • Deeper Graphify coupling, harder to maintain

My Recommendation: B with ghost anchors for top 10 concepts

Q&A format maps directly to how agents query. Ghost knowledge for the 10 most critical architectural concepts (search pipeline, graph layers, hook system, etc.) ensures zero misses on things that matter most. Confidence decay on the Q&A entries handles staleness without manual cleanup.
