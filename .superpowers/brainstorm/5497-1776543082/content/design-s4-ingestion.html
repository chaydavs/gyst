Design: Section 4 — MD Ingestion + Self-Document Command

How markdown files flow into the KB, and what gyst self-document does end-to-end

MD File Ingestion Pipeline (runs automatically)

Trigger 1 — FileChanged hook

Watches **/*.md. Any markdown file saved on disk fires this hook immediately. No polling, no CLI command needed. The hook script calls gyst emit md_changed --file <path> which queues the file for ingestion.

Trigger 2 — InstructionsLoaded hook

When Claude loads CLAUDE.md or any .claude/rules/*.md file, gyst auto-ingests it. Provides file_path + memory_type — stored with high confidence (0.95) since these are intentionally written guidelines.

Trigger 3 — gyst self-document / codebase scan

Glob scan for all *.md files. Hash-check each against source_file_hash stored in DB — skip if unchanged, reingest if modified. Runs on install and on every git pull hook.

What gets extracted from each MD file

Title → KB entry title
H2/H3 headings → section summaries
Frontmatter tags → KB tags
File path → source_file pointer
Named entities → graph edges to structural nodes
Code blocks → pattern extraction
Decision headers → auto-type as decision
File hash → change detection

gyst self-document — 3-phase bootstrap

Phase 1 — Structural Skeleton (zero tokens)

Run Graphify AST on the entire codebase. Every module, function, and import relationship → KB entries (type: structural). Build the initial call graph edges. ~2s for a medium codebase.

Phase 2 — MD Corpus Scan (zero tokens)

Glob all *.md files, parse + ingest. Stores source_file pointer on every entry. Links to structural nodes via entity match. Ingests: README, CLAUDE.md, docs/**, decisions/**, specs/**, ARCHITECTURE.md, any plan docs.

Phase 3 — Ghost Knowledge Generation (Haiku, token-efficient)

Compute graph centrality. Top 10 nodes → one Haiku call each. Input: structural entry + linked MD docs + git blame summary. Output: a concise ghost entry ("How does [concept] work?") with infinite confidence. Total cost: ~10 Haiku calls × ~500 tokens = ~5000 tokens for the entire bootstrap.

End state: what an agent experiences

Agent calls recall("how does the search pipeline work") → gets: ghost knowledge entry (always tier 0) + structural node for store/search.ts + ARCHITECTURE.md excerpt with the search section + any session learnings about RRF tuning. Zero source file reads needed.
