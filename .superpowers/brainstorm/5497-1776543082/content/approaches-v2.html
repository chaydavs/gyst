Revised: Zero Manual Work

All three approaches are fully automated — the question is what drives the generation engine

A

Graphify → KB Promotion

Gyst already builds a structural AST graph of your codebase. Auto-promote those nodes into KB entries. Each module/function/pattern becomes a KB entry with its dependents and callers as graph edges. No LLM needed — pure structural analysis.

Pros

  • Zero tokens — no LLM call to generate
  • Always structurally correct (derived from actual code)
  • Rich graph edges = traversal chains work perfectly
  • Runs on every git commit, zero config

Cons

  • Entries describe WHAT, not WHY or HOW
  • "src/store/search.ts exports: searchKB()" doesn't answer "how does search work"
B

Harvest-at-Depth ★ Recommended

During every session, whenever an agent reads a source file or makes an architectural decision, the PostToolUse hook extracts a KB entry automatically — title, content, type, confidence, and graph links inferred from context. No human writes anything. Entries are generated from actual agent actions, not speculative questions.

Plus a one-time bootstrap: gyst self-document runs a structured pass over the codebase and writes module summaries using git history + docstrings + comments as source material. Haiku-class LLM call, very token-efficient.

Pros

  • Entries are exactly what agents actually needed — guaranteed relevance
  • Bootstrap is one Haiku call per module, not per line
  • Self-healing: every agent session improves the KB
  • Git hooks capture "why" from commit messages automatically

Cons

  • KB starts sparse — first few sessions hit cold KB
  • Bootstrap LLM call required for initial coverage
C

A + B Hybrid: Structural Skeleton + Harvest Flesh

Graphify gives the skeleton (modules, relationships, call graph) — instant, zero tokens. Harvest adds the flesh (explanations, decisions, failure patterns) over time. Ghost knowledge for the top 10 most-connected nodes in the graph.

Pros

  • No cold-start problem — structural skeleton exists day one
  • Graph traversal works immediately from structural edges
  • Harvest enriches over time without any manual effort
  • Most token-efficient long-term (structural = free, harvest = amortized)

Cons

  • More complex to build (two pipelines)
  • Need to avoid duplicate entries (structural + harvested for same concept)

My Recommendation: C (Hybrid)

Graphify gives you instant structural coverage on day one with zero tokens. Harvest-at-depth fills in the WHY over time. Ghost knowledge for the 10 highest-centrality nodes guarantees those always surface. No human writes a single entry.
