Design: Section 3 — Hook Capture Expansion

Gyst currently uses 6 hooks. These 6 additional hooks unlock far richer context capture.

Currently wired (6 hooks)

SessionStart UserPromptSubmit PreToolUse PostToolUse Stop SubagentStop
★

PreCompact → Harvest before context is lost

Highest value new hook. Fires right before context compaction erases conversation history. Gyst reads transcript_path (available on every hook) and runs a full harvest pass before the context window is cleared. This is the only moment where the full session context is still intact.

KB entries created: learnings, decisions, architecture notes from the full conversation

2

InstructionsLoaded + FileChanged → Auto-ingest MD files

InstructionsLoaded fires when CLAUDE.md or .claude/rules/*.md load — provides file_path and memory_type. Auto-ingest these immediately as MD doc entries (Layer 2).

FileChanged can watch any file pattern. Configure it on **/*.md to auto-reingest any markdown file the moment it changes on disk — plans, specs, ADRs, architecture docs, all ingested instantly without any CLI command.

KB entries created: md_doc type, source_file pointer, auto-linked to related structural nodes

3

PostToolUseFailure → Auto-extract error patterns

Fires when any tool fails with an error. Provides tool_name, tool_input, and error. Run gyst's error normalizer on the error string and auto-create an error_pattern KB entry — no agent involvement needed. Real failures from real sessions become searchable patterns immediately.

KB entries created: error_pattern type, normalized signature, linked to affected file

4

PreToolUse(Read) → Detect KB misses in real time

When an agent calls Read on a source file, that's a signal the KB failed — the agent couldn't get what it needed from recall(). Capture: which file was read, what the preceding prompt was (from transcript_path), and queue a documentation job for that file. After the session, those queued files get auto-documented first.

Side effect: KB miss rate becomes measurable — feeds directly into drift score

5

SubagentStart → Inject ghost knowledge into every subagent

Fires when any subagent spawns, provides agent_type. Return additionalContext with the ghost knowledge + top conventions for that agent's type. Currently, subagents start cold — they don't inherit the parent's gyst context. This gives every spawned agent immediate architectural awareness.

Impact: subagents stop reading CLAUDE.md repeatedly; ghost knowledge lands in their context directly

6

PostCompact → Verify harvest completed, snapshot drift

Fires after compaction. Verify the PreCompact harvest landed successfully, then take a drift snapshot so the new (post-compaction) session starts with a fresh baseline. Lightweight — just a confirmation + gyst status ping.

Ensures compaction never silently loses knowledge

Net result: 12 hooks total, full lifecycle coverage

With these additions: gyst captures at every meaningful event — session start, prompt submit, every tool call, every failure, every file change, every MD load, every compaction, every subagent lifecycle. The knowledge graph updates in real time as work happens, not just at session end.
